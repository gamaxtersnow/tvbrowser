# Apple TV 国内视频平台浏览器

## 问题定位
优酷、腾讯视频、爱奇艺均无 tvOS 客户端，但其网页版播放器功能已完整（认证、播放、DRM、选集等均已解决）。用户在国内，持有三家平台 VIP 会员，需要一个适配遥控器操作的 tvOS 浏览器，直接使用网页版观看。

## 需求

**浏览器核心**
- R1. 基于 WKWebView 加载三家平台的网页版（youku.com / v.qq.com / iqiyi.com）
- R2. 支持用户在各平台网页内完成会员登录，登录态（Cookie）持久化
- R3. 提供按平台的入口（首页 Tab 或快捷入口），进入后即为对应网站的完整浏览体验

**遥控器适配**
- R4. Siri Remote 触摸板 / 方向键映射为页面焦点移动和滚动
- R5. 点击操作映射为网页元素点击，播放时映射为播放/暂停
- R6. Menu 键优先关闭当前浮层或返回上一页，无历史页面时退出 App

**播放体验**
- R7. 自动检测并全屏播放网页视频（HLS），无需手动触发全屏
- R8. 视频全屏时，遥控器方向键映射为快进快退（每次 10 秒）和音量控制

**TV 端视觉适配**
- R9. JavaScript 注入：放大字体、调整列表间距，适配电视 3-5 米观看距离
- R10. 注入脚本隐藏移动端的顶部导航、底部 Tab 栏等非内容 UI 元素

## 成功标准
- 能够用遥控器在三个平台间切换
- 能够用遥控器浏览内容并点击播放
- 视频全屏状态流畅播放 1080p 以上
- 登录一次后 Cookie 保持，再次打开无需重新登录

## 范围边界
- 不做后端服务 —— 纯 tvOS 客户端
- 不做公开分发（不上架 App Store、不用 TestFlight）
- 不做视频下载或离线缓存
- 不处理直播内容（仅点播）
- 不做弹幕、评论等社交功能
- 不做统一聚合搜索（直接使用各平台自己网页的搜索）

## 关键决策
- **纯 tvOS WKWebView 浏览器方案**：直接复用网页版，后端和 API 逆向往复的复杂度由平台自己承担
- **按平台分别入口**：用户习惯分别浏览各平台内容库
- **JavaScript 注入适配 TV 交互**：不在网页功能上重复造轮子，只做遥控器和视觉层面的适配

## 依赖与假设
- 用户在国内，可直接访问三家平台的网页和 CDN
- 用户持有各平台的有效 VIP 会员
- 三家平台的网页版播放器使用 HLS（WKWebView 原生支持）
- tvOS WKWebView 的 JavaScript 注入能力足以完成焦点导航和 UI 优化
- 平台网页结构可能变化，JS 注入脚本需跟随网站更新维护

### 推迟到规划阶段
- [影响 R4-R5][需要调研] tvOS WKWebView 对 focus()/blur() 和键盘事件的支持程度，Siri Remote 按键事件如何映射到网页交互

- [技术选型] 最低 tvOS 版本兼容性选择
